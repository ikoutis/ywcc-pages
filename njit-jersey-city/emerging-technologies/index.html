Course focus

Build the judgment layer above the tools.

Students work across the full software lifecycle while learning where generative AI accelerates engineering work, where it creates risk, and how to prove the difference.

From prompt to precise intent

Write specs that AI systems can execute against.

Practice requirements elicitation, user stories, acceptance criteria, type signatures, invariants, and formal contracts that reduce ambiguity before code generation begins.

Human-in-the-loop agent design

Coordinate AI coding assistants like an engineering lead.

Explore ReAct, Reflexion, tool use, memory, repository context, and multi-agent workflows using production model APIs and emerging software engineering frameworks.

Evidence before confidence

Build safety nets for AI-generated code.

Combine type checking, unit tests, property-based tests, fuzzing, static analysis, mutation scores, and targeted formal reasoning into a layered validation pipeline.

Professional delivery

Release a working system with traceable AI contributions.

Teams deliver a sophisticated software system with CI gates, security review, deployment, technical reporting, live demonstration, and a provenance ledger for AI-assisted work.

Semester project

The project is the spine.

Teams of three or four build, review, and maintain an industrial-scale software system primarily through AI-assisted development.
